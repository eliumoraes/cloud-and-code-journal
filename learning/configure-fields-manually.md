# Guia Rápido: Configurar Priority e Sprint no Project Board

## Método Rápido (Recomendado)

### Passo 1: Abrir o Project Board
1. Acesse: https://github.com/eliumoraes/cloud-and-code-journal/projects
2. Abra o board "Cloud and Code Journal"

### Passo 2: Configurar em Lote (Múltiplas Issues)

1. **Selecione múltiplas issues**:
   - Clique e arraste para selecionar várias issues
   - Ou use Ctrl+Click para selecionar issues individuais

2. **Configure campos em massa**:
   - Com as issues selecionadas, clique em qualquer campo Priority ou Sprint
   - Selecione o valor (ex: High, Medium, Low para Priority)
   - Digite o valor (ex: Week 1, Week 2, etc. para Sprint)
   - O valor será aplicado a todas as issues selecionadas

## Método Individual

Se preferir configurar uma por uma:

1. Clique no card da issue no board
2. Clique no campo **Priority** e selecione: High, Medium ou Low
3. Clique no campo **Sprint** e digite: Week 1, Week 2, Week 3 ou Week 4
4. Repita para cada issue

## Dica: Filtrar por Label

Para facilitar, você pode filtrar o board por label:
- **Learning**: Issues de estudo
- **Challenge**: Issues de coding challenges
- **Project**: Issues de POCs
- **Journal**: Issues de journal

Isso ajuda a encontrar e configurar issues relacionadas mais rapidamente.

## Verificação

Após configurar, verifique:
- Todas as issues têm Priority definida
- Todas as issues têm Sprint definida
- Issues da mesma semana têm o mesmo Sprint
- Priorities estão corretas conforme tabelas acima
